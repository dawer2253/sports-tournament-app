# Obiekty (Admin): specyfikacja

Ekran administracyjny z listą obiektów (boisk, hal), na których rozgrywane są mecze turnieju.

<Canvas of={Stories.Domyslny} />

## Cel
Umożliwić organizatorowi przeglądanie i zarządzanie miejscami rozgrywek: dodawanie, edycję i usuwanie obiektów przypisywanych później do meczów w terminarzu.

## Dane (wejście)
- Lista obiektów: `name` (nazwa), `address` (adres), `matches` (liczba przypisanych meczów).
- Źródło: `venues` z warstwy danych (docelowo z API turnieju).

## Reguły
- Kolumna „Mecze" pokazuje liczbę meczów przypisanych do obiektu (`tabular-nums`).
- Nazwa obiektu prezentowana z ikoną `MapPin` i pogrubieniem; adres w kolorze `text-muted-foreground`.
- Akcje wiersza (edycja, usunięcie) wyrównane do prawej, ikonograficzne.

## Stany
- **Lista obiektów**: domyślny (powyżej).
- **Brak obiektów**: pusta tabela z zachętą „Dodaj obiekt" (TODO).
- **Usuwanie z przypisanymi meczami**: potwierdzenie z ostrzeżeniem, gdy `matches > 0` (TODO).
